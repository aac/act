# Project skeleton and repo layout

## Context
This is the very first issue in the build DAG; every other issue assumes a
predictable on-disk layout. The spec (§Data model and storage) describes the
runtime `.act/` layout but says nothing about source-tree organization. We fix
that here so subsequent issues can drop code into stable locations without
bikeshedding.

## Scope
- Top-level directories: `cmd/act/` (the binary entry point), `internal/` (all
  non-exported packages), `docs/`, `testdata/`.
- Stub `internal/` sub-packages: `canonicaljson`, `hlc`, `ids`, `op`, `store`,
  `fold`, `index`, `cli`, `config`. Each gets a `doc.go` declaring the package
  with a one-line description matching the spec section it implements.
- `.gitignore` covering build artefacts and the gitignored `.act/index.db`.
- `Makefile` with `build`, `test`, `lint`, `fmt`, `tidy` targets (the targets
  may shell to `go` once act-9cad lands; for this issue they may be no-ops or
  echo placeholders).
- `README.md` listing the layout and pointing at `docs/spec-v2.md`.

## Out of scope
- Actual `go.mod` / `go.sum` (act-9cad).
- CI configuration (act-9cad).
- Any executable Go logic.

## Implementation notes
- Use `internal/` (not `pkg/`) — every package is private to the binary.
- Package names are lower-case, single-word; no underscores.
- `doc.go` body is one `// Package <name> ...` sentence followed by `package <name>`.
- `.gitignore` MUST include `.act/index.db` because the spec (§On-disk layout)
  requires the writer to add it on `act init`; we keep the repo's own dogfood
  `.act/` clean too.
- The Makefile uses tabs (real tabs) for recipe indentation.

## Test plan
- Add a CI smoke check in act-9cad that runs `make build` and asserts exit 0.
- A repo-layout test in `internal/cli` (added later) asserts the package import
  graph stays inside `internal/`.
- Manual: `git ls-files | sort` snapshot committed under `testdata/layout.txt`
  for human review; not a regression gate yet.
