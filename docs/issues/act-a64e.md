# Property tests and fuzzer

## Context
Implements spec-v2 §7.1 (property tests) and §7.3 (fuzz). The fold engine's correctness rests on permutation invariance (commutative-disjoint ops yield identical state regardless of order) and HLC monotonicity (single-node sequences strictly increase in HLC even under local-clock skew). The fuzzer pins fold determinism: cold and warm fold of the same op sequence produce byte-identical canonical JSON.

## Scope
- Two property tests:
  - `prop_fold_permutation_invariance` — generator emits commutative-disjoint op sets; every permutation is folded; all permutations must yield byte-equal canonical-JSON state.
  - `prop_hlc_monotonicity` — generator emits ops from a single simulated node with an injectable clock that includes backward jumps; HLC strictly increases across the emitted sequence.
- One fuzzer:
  - `fuzz_fold_determinism` — random op-sequence generator with shrinking; folds cold (fresh state) and warm (incremental); asserts byte equality.
- Corpus committed under `testdata/fuzz/`; the test harness reads the corpus, runs each, and adds new minimized failures back to the corpus.

## Out of scope
- Golden tests (act-9b55).
- Concurrency tests (act-0c76).
- CI matrix wiring (act-2e8d).
- Property tests for commands beyond fold semantics (e.g., CLI flag parsing).

## Implementation notes
- Generator API: `GenerateOpSet(rng, params) []Op` and `GenerateOpSequence(rng, params) []Op`. Both produce well-formed envelopes with valid HLC and node_id. Disjointness is enforced by partitioning issue_ids and field names per generated op.
- Permutation enumeration is bounded: for sets larger than N=8 the test samples K=200 random permutations rather than exhausting; the seed is logged so failures are reproducible.
- The fuzzer's shrinker is delta-debugging style: bisect the sequence repeatedly, retaining the shortest failing prefix/subsequence.
- Cold vs. warm fold: cold = empty state then apply all ops; warm = apply first half, persist via the checkpoint path (act-a1f6 hooks), restart, apply rest. Byte equality is asserted on canonical JSON of the final state.
- Backward-clock generator: the injectable clock returns a sequence of monotonically chosen times with occasional negative deltas; HLC's logical counter must absorb the regression.

## Test plan
- Cite spec §7.1 / §7.3.
- Assert generator coverage: a smoke test confirms the generator produces all op-types over 1000 draws.
- Reproducibility: a known-seed run produces identical generated bytes across two invocations.
- Regression-add behavior: simulate a fuzz failure via a planted bug; assert the minimized case is appended to `testdata/fuzz/` and the next run reproduces the failure deterministically.
