# act log command

## Context
Spec §3 `act log <id>` (lines 760–774). Read-only enumeration of an issue's op files (act-6ec9 file naming) sorted by HLC. No fold; this is the raw audit log.

## Scope
- Parse positional `<id>` and flag `--json`.
- Resolve id via the pipeline.
- Read all op files under `.act/ops/<id>/**` and parse envelopes.
- Sort by HLC (timestamp_ms asc, logical asc, node_id asc).
- Emit each op's metadata and full payload.

## Out of scope
- Folding; the raw stream is reported regardless of merge/redaction outcomes (redacts are part of the stream too).
- Filtering by op type or HLC range (future enhancement).
- Mutation.

## Implementation notes
- Flags:
  - `--json` bool.
- Exit codes:
  - `0` success.
  - `2` ambiguous prefix.
  - `3` `.act/` missing or unknown id (per §5.C.1).
  - `4` writer-version skew (any op carries `writer_version > self`).
- JSON schema:
  ```
  {"ok": true,
   "id": "act-<16hex>",
   "ops": [
     {"op_id": "<hex>", "op_type": "create|update_field|close|...",
      "hlc": "<rfc3339-with-logical-and-node>",
      "node_id": "<8hex>", "writer_version": "<semver>",
      "payload": {...},
      "path": ".act/ops/act-<id>/<yyyy-mm>/<filename>.json"}
   ]}
  ```
- HLC string MUST round-trip exactly (per §5.A.3) — emitted verbatim from the op envelope.
- `path` is repo-relative; consumers can resolve against the repo root.
- Side effects: none (read-only).
- Skew detection: scan all ops for `writer_version > self.writer_version`; if any, exit 4 before emitting partial output.

## Test plan
- Issue with create + 2 updates + close: 4 ops returned in HLC order.
- Resolve via shortest unique prefix.
- Ambiguous prefix: exit 2.
- Unknown prefix: exit 3.
- `.act/` missing: exit 3.
- Op with `writer_version > self`: exit 4.
- Verify op_id matches the file's recorded `op_id` (sha256 of canonical payload per §5.B.1).
- Verify HLC string round-trips byte-exact.
- Multi-month op layout (`<yyyy-mm>` subdirs): all months traversed, sort still global by HLC.
- Issue containing a `redact` op: redact appears in stream (raw log is unfiltered).
- Concurrent ops with equal timestamp_ms but different logical/node_id: deterministic ordering.
- JSON object key order stable; consumer can parse back into the same envelope.
