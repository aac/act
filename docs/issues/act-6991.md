# Shortest-unique-prefix display and resolution

## Context
Spec §ID model specifies git-style shortest-unique-prefix display: per
invocation, the CLI computes the shortest prefix that uniquely identifies
each known issue and uses that for `act show`, `act list`, `act ready`,
`act log`. The reverse direction — resolving a user-supplied prefix into
a full id — is a strict pipeline shared by every command (§Pre-import id
resolution). This issue centralizes both algorithms in `internal/ids`.

## Scope
- `internal/ids` extension:
  - `ShortestUniquePrefixes(full []FullID) map[FullID]ShortID`.
  - `Resolve(input string, known []FullID) (FullID, error)`.
  - `ErrAmbiguousID` type carrying `Candidates []FullID`.
  - `ErrNotFound` sentinel.
- Pure functions; no disk access.

## Out of scope
- Loading the set of known full ids from `.act/ops/` (caller's job, via
  fold or a directory scan).
- Import-map rewriting (act-6eff and per-command resolvers).
- CLI rendering (per-command issues; this only supplies the strings).

## Implementation notes
- Algorithm: build a trie over the hex tails (after stripping `act-`).
  For each leaf, walk back to the deepest branching node; the depth
  + 1 is the unique-length, with a floor of 4 hex.
- Tie-breaking is purely lexicographic on the hex; no HLC involvement.
- `Resolve` strips an optional `act-` prefix, lowercases, then matches
  against the hex tails. Returning `ErrAmbiguousID` populates up to 16
  candidates (truncate silently; the user only needs enough to retype).
- Candidate order for `ErrAmbiguousID` is lexicographic; tests rely on it.
- Inputs shorter than 1 hex char (just `act-` or empty) error
  `ErrNotFound` for symmetry with the zero-match case.

## Test plan
- Unit: a 6-element set with one prefix collision yields exactly two
  longer short ids and four 4-hex short ids.
- Unit: `Resolve("a1b2", [...])` returns the unique match; `Resolve("a")`
  on a set with multiple matches errors with all candidates listed and
  sorted.
- Unit: case-insensitivity check (`Resolve("A1B2", ...)` works).
- Property test: random sets of 1..1000 full ids; for each, every
  resulting short id resolves uniquely back to its full id.
- Edge case: a single full id in the set yields a 4-hex short id (the
  spec floor), even though 1 hex would technically suffice.
- Edge case: two full ids that share their entire 40-hex prefix (i.e.
  identical) are an upstream invariant violation; tested only that the
  function does not panic and returns one entry per unique id.
