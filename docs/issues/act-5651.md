# act update command

## Context
Spec §3 `act update <id>` (lines 639–669). Generates one or more field-update ops per invocation; supports the atomic claim protocol (§2.4, act-9824). Exit-code clarifications per §5.C.2 and §5.C.3.

## Scope
- Parse positional `<id>` and field flags: `--status`, `--priority`, `--assignee`, `--description`, `--accept` (repeatable), `--dep-rm` (repeatable), `--claim`, `--wait`, `--wait-timeout`, `--json`, plus universal write flags.
- Resolve `<id>` and any `--dep-rm` ids via the pipeline.
- Reject `--status closed` immediately (§5.C.2).
- For each field flag, emit one op; commit all ops together (single git commit).
- For `--claim`: drift-check first (§5.C.3), then write `claim` op, fetch+pull-rebase (unless `--isolated`), refold, evaluate winner.
- Honor `--wait`/`--wait-timeout` only with `--claim`.

## Out of scope
- Closing issues (delegated to `act close`).
- Adding dep edges (delegated to `act dep add`).
- Reopening (separate op type, future command).

## Implementation notes
- Flags:
  - `--status` enum `open|in_progress|blocked` only; `closed` exits 2 per §5.C.2.
  - `--priority` int 0..3.
  - `--assignee` string; empty string clears.
  - `--description` string.
  - `--accept` repeatable; each appends one criterion (uses `update_field` semantics over the criteria list).
  - `--dep-rm <id>` repeatable; resolves id; removes the edge; missing edge exits 1.
  - `--claim` bool.
  - `--wait` bool, requires `--claim` (else exit 2).
  - `--wait-timeout SECS` int, default 60.
  - `--json` bool.
  - Universal write flags apply.
- Exit codes:
  - Claim mode: `0` win, `1` loss or other logical failure (drift, dep-rm-missing), `2` usage.
  - Non-claim mode: standard universal exit codes; `--status closed` always exit 2.
  - `3` `.act/` or id missing; `4` writer-version skew.
- JSON outputs: claim win/loss as in spec; non-claim writes return `{"ok": true, "id": "...", "ops_written": [<op_types>], "committed": <bool>, "pushed": <bool>}`.
- Side effects: 1..N op files in `.act/ops/<id>/...`, single git commit (unless `--no-commit`), optional fetch+pull-rebase under `--claim`, optional push.
- HLC drift check: read repo reference (max HLC across recent ops); if `|now - ref| > 5min`, exit 1 BEFORE network mutation (§5.C.3).
- `--claim` with no remote configured: stderr warning, proceed local-only.

## Test plan
- `--status in_progress --priority 0`: two ops, one commit, `ops_written` lists both.
- `--status closed`: exit 2 unconditionally regardless of current state.
- `--assignee ""`: clears assignee.
- `--accept "x" --accept "y"`: appends two criteria in order.
- `--dep-rm <existing>`: edge removed, exit 0.
- `--dep-rm <missing>`: exit 1 (`{"error":{"kind":"dep_not_found"}}`).
- `--claim` with fresh issue: claim op written, win reported, exit 0.
- `--claim` losing race: exit 1, JSON shows winner node_id.
- `--claim` with HLC drift >5min: exit 1 BEFORE any network call (verify via stub).
- `--wait` without `--claim`: exit 2.
- `--wait` with claim loss, polling until releasable: succeeds within timeout.
- `--wait` exhausting `--wait-timeout`: exit 1.
- `--isolated` + `--claim`: skips fetch/pull-rebase, evaluates locally.
- `--no-commit --push`: exit 2.
- Ambiguous id prefix: exit 2.
- Unknown id: exit 3.
