# On-disk layout and .act/config.json

## Context
Spec Â§On-disk layout and Â§1.2 (Persistence) define the exact directory tree
under `.act/` and the persistence semantics for `last_hlc`. Every command
that reads or writes ops needs both (a) consistent path derivation and
(b) atomic config updates so that a crash mid-write leaves a parseable
state. We isolate this in `internal/store` and `internal/config` to keep
later command code from open-coding paths.

## Scope
- `internal/store/layout.go`: path discovery and constants. Exported helpers
  return cleaned absolute paths.
- `internal/store/lock.go`: cross-platform advisory lock around `.act/.lock`,
  used during id-collision protocol and op writes.
- `internal/config/config.go`: typed read/write of `.act/config.json` via
  `canonicaljson`.
- Default-value constructor for fresh installs (used by `act init`,
  act-b0b9): `schema_version=1`, `auto_commit=true`, `auto_push=false`,
  `hlc_drift_budget_seconds=300`, `last_hlc.wall=0`, `last_hlc.logical=0`.

## Out of scope
- `node_id` derivation from machine-id/email (act-b0b9).
- Creating the `.act/` tree on `act init` (act-b0b9).
- SQLite index file lifecycle (act-912f).
- Fold checkpoint reading (act-a1f6).

## Implementation notes
- `Layout.Discover` walks `cwd` upward via `filepath.Dir` until it finds a
  `.git` entry (file or directory â€” submodules show a file). Bare repos
  (no working tree) error with `ErrNotInGitRepo`.
- Atomic writes use `os.CreateTemp` in the same directory, `f.Sync()`, then
  `os.Rename`. `Sync` the parent directory afterward on POSIX.
- Locking wrapper: build-tagged `lock_unix.go` and `lock_windows.go`.
  Both expose `Lock(path) (Unlocker, error)` with `Unlocker.Close() error`.
- The lock helper does NOT spin; callers wrap in retry if needed.
- All paths returned are `filepath.Clean`ed and absolute.
